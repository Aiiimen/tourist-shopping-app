Screen padding

16px horizontal padding on 375px screens. Cards: 16px internal padding. Section separators: 24px vertical gap.

Touch targets

44px minimum height for all interactive elements. Primary buttons: 48px tall. Bottom nav icons: 56px touch area. Map pins: 44×44px invisible touch zone centred on the pin.

Bottom sheet

Shopping list slides up as a bottom sheet. Collapsed: 80px peek (shows first 1–2 items). Expanded: 70% screen height. Map always visible behind collapsed sheet.

Button States
Primary (blue) = main page action. Accent (tangerine) = floating action button / "Add Item" (one per screen). Success (jade) = purchase confirmation. Outline = secondary actions. Ghost = tertiary. Danger = destructive actions (require confirmation).
06 — Design Principles
Non-negotiable Constraints
☀
Outdoor First
All text colours meet WCAG AA (4.5:1) minimum; body text targets AAA (7:1). Blue-800 on white = 12:1. Tested against simulated glare.
📱
375px Baseline
Every spacing, font size, and layout decision starts at 375px. Scale up for larger screens. Design one-handed from the start.
🗺
Map is the Hero
60–70% of screen space belongs to the map. UI overlays via translucent bottom sheets and floating chips — never competes with the map.
🌎
Language-Agnostic
Icons take precedence over labels in UI chrome. Category icons are internationally understood. No idioms or locale-specific phrasing in the interface.
🌈
Colour Codes Items
Each item gets a persistent category colour — same on list, map pins, and store cards. Builds a visual vocabulary the user learns in <30 seconds.
✅
Progress is Satisfying
Buying something is the goal. Purchased state = immediate, satisfying visual change: jade tick, strikethrough, pin greyed out. Don't understate success.
